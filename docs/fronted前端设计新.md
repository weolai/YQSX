# YQSX 智能零食商城最终项目文案规范

## 0. 文案总原则

### 产品定位

YQSX 是一个面向零食购物场景的智能商城系统，结合拍照识别、个性化推荐、智能问答与在线下单能力，帮助用户更快找到想吃的零食，并完成稳定、可信的购买流程。

### 前台表达原则

| 原则         | 说明                                                   |
| ---------- | ---------------------------------------------------- |
| 少讲技术，多讲价值  | 前台用户不需要理解 DIN、YOLO、KBOQA 等技术细节，只需要知道系统能帮他识别、推荐、下单。   |
| 不暴露系统内部状态  | 不在用户界面直接出现“降级、限流、Feign、Gateway、Trace ID、后端控制台”等开发语言。 |
| AI 不过度承诺   | 拍照识别和智能推荐需要提示“结果可能受图片清晰度、历史行为等因素影响”。                 |
| 支付与订单要专业可信 | 支付、订单、账号相关页面避免过度可爱化，优先表达安全、确认、状态清晰。                  |
| 错误文案要给下一步  | 错误提示必须告诉用户可以怎么做，例如“重新加载、返回首页、稍后重试”。                  |

### 禁用词

以下词汇不得出现在生产环境用户界面中：

```text
演示环境
测试账号
后端控制台
Gateway
Feign
限流
降级
Trace ID
服务熔断
Nacos
Sentinel
YOLO 模型异常
DIN Top40 纯数字商品 ID
```
