- 微服务架构

### 配图建议

- 唯一索引：用户名、手机号、订单号

- 数据隔离：各服务独立数据库

### 配图建议

### 演讲稿
数据库设计上，我坚持微服务数据库隔离原则，每个服务维护自己的数据，避免跨库直接访问。核心表覆盖用户、商品、订单、支付和 AI 相关日志，同时为高频查询字段设计了索引。这样既满足业务需要，也保证了查询效率和后期维护性。

- SQL 注入 / XSS / CSRF 防护
- 数据库隔离

### 配图建议

- 优化：缓存、索引、分页、预计算

### 配图建议

### 演讲稿
项目里最大的难点主要有三个：一是识别结果怎么准确映射到商品；二是订单和支付的多服务协同要保证一致性；三是推荐系统既要准又要快。针对这些问题，我分别采用了类别映射、幂等控制、召回加精排和缓存预计算的方式去优化，整体效果比较稳定。
