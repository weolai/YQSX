# 项目概述

## 1.1 项目背景

随着电子商务和人工智能技术的快速发展，传统电商平台面临商品检索效率低、用户选择困难、购物体验单一等问题。消费者在面对海量商品时，往往需要花费大量时间浏览和筛选才能找到心仪商品。与此同时，计算机视觉、深度学习推荐算法和知识图谱技术的成熟为解决这些问题提供了新的技术路径。

本项目——**YQSX智能零食商城**，旨在构建一个融合AI图像识别、个性化推荐和知识图谱问答的新一代智能电商平台，实现"拍照即购物"的创新购物体验。

## 1.2 项目目标

### 业务目标

- 提供基于图像识别的智能商品检索功能，降低用户搜索成本
- 通过个性化推荐算法提升用户转化率和复购率
- 利用知识图谱问答提供智能客服体验，解答用户商品咨询
- 构建高可用、可扩展的微服务架构，支撑业务增长

### 技术目标

- 采用Spring Cloud Alibaba微服务架构，实现服务解耦与独立部署
- 集成YOLOv11目标检测模型，实现19类零食商品的实时识别
- 基于DIN（Deep Interest Network）深度兴趣网络实现个性化推荐
- 构建基于Neo4j的商品知识图谱，支持自然语言问答
- 实现完整的可观测性体系（链路追踪、指标监控、日志管理）

## 1.3 核心功能

### 1.3.1 AI拍照识别购物

用户上传零食图片，系统通过YOLOv11模型自动识别商品类别，匹配对应商品并一键加入购物车，实现"所见即所得"的购物体验。

- 支持19类零食商品识别（Ashi Mashi、Chee、Cheetoz、Maz Maz、Lay's、Doritos等）
- 识别准确率达到98%以上
- 平均推理响应时间150ms（CPU）/ 50ms（GPU）

### 1.3.2 个性化商品推荐

基于DIN深度兴趣网络，根据用户历史行为序列（浏览、点击、收藏、购买）动态计算用户兴趣，实时生成个性化推荐列表。

- 支持用户兴趣激活与注意力加权
- 召回+精排两阶段推荐架构
- AUC指标达到0.85+
- 推荐响应时间<100ms（缓存命中）

### 1.3.3 知识图谱智能问答

构建商品知识图谱，支持自然语言提问，如"推荐一款辣味薯片"、"哪些商品是无糖的"，通过实体识别、关系抽取和Cypher查询返回精准答案。

- 基于Neo4j图数据库存储商品知识
- 支持中文实体识别与链接
- 预置Cypher查询模板
- 问答准确率90%+

### 1.3.4 完整电商功能

- 用户注册登录与JWT认证
- 商品浏览、搜索、分类筛选
- 购物车管理与订单创建
- 在线支付与订单状态跟踪
- 个人中心与订单管理

## 1.4 技术创新点

### 1.4.1 多模态AI融合

创新性地将计算机视觉（YOLOv11）、序列推荐（DIN）和知识图谱（Neo4j KBQA）三种AI技术融合到同一电商平台，形成"视觉识别→兴趣推荐→知识问答"的完整AI能力闭环。

### 1.4.2 两阶段推荐架构优化

在DIN推荐系统中，采用"召回+精排"两阶段架构：
- 召回阶段：基于用户历史行为类目关联+全局热门商品，从36万商品中快速筛选1万候选
- 精排阶段：使用DIN模型对候选集进行精准打分
- 性能优化：内存峰值从600MB降至12MB，推理耗时从数十秒降至秒级

### 1.4.3 缓存安全加固

推荐系统缓存采用JSON+HMAC签名方案替代传统pickle序列化，消除任意代码执行（RCE）安全风险，同时保证缓存完整性校验。

### 1.4.4 微服务可观测性

完整集成Sleuth+Zipkin链路追踪、Prometheus+Grafana指标监控，实现全链路可观测，问题定位效率提升80%。

- 提供知识图谱问答在电商客服场景的实践方案

### 工程价值

- 提供完整的Spring Cloud Alibaba微服务最佳实践
- 包含AI模型服务化（Python+FastAPI与Java微服务集成）的工程方案
- 提供从开发到部署的完整可观测性方案

### 应用价值

- 提升用户购物效率，降低搜索成本
- 通过个性化推荐提升平台GMV
