# 基于微服务架构和多模态 AI 的智能零食电商系统设计与实现

## 摘要

[Confirmed] 本文围绕 YQSX 智能零食商城的设计与实现展开，系统融合微服务架构、YOLOv11 图像识别、DIN 深度兴趣推荐与 Neo4j 知识图谱问答能力，形成“拍照识别—个性推荐—智能问答”的一体化购物流程。系统采用前后端分离与服务自治设计，覆盖商品浏览、订单管理、支付处理、AI 识别、推荐与问答等核心业务场景。

[Confirmed] 论文初稿基于已确认的 research 结果整理，文中凡涉及性能指标均优先标注来源；若对应位置无法确认真实测试日志，则仅保留测试目标或预期指标，不将其写作已验证结果。全文统一使用 YOLOv11 表述。

**关键词**：微服务；YOLOv11；DIN；知识图谱；智能电商；YOLOv11

## 第1章 绪论

### 1.1 研究背景

[Confirmed] 随着电子商务平台商品规模持续扩大，传统基于关键词的检索方式在效率、精准度和个性化体验方面逐渐暴露出不足。用户在面对大量零食商品时，往往需要经过多轮搜索、筛选与比较，才能定位目标商品，购物路径较长，体验较为分散。

[Confirmed] 计算机视觉、推荐系统与知识图谱技术的发展，为电商系统升级提供了可落地的技术路径。基于图像的商品识别能够降低用户搜索成本，基于行为序列的推荐能够提升商品分发效率，基于知识图谱的问答能够增强商品咨询与导购能力。因此，构建一个融合多模态 AI 能力的智能零食电商系统具有明确的工程与应用价值。

### 1.2 研究意义

[Confirmed] 本课题的工程意义在于：一方面，通过微服务拆分实现业务解耦，提高系统可维护性与扩展性；另一方面，通过引入 YOLOv11、DIN 与 KBQA 等 AI 能力，增强电商系统的智能化水平，提升用户从识别、推荐到咨询的整体体验。

[Confirmed] 本课题的应用意义在于：在垂直零食电商场景中，系统可以支持拍照识物、个性推荐与智能导购，减少用户的检索与决策成本，并为后续扩展更多商品类目提供通用架构基础。

### 1.3 国内外研究现状

[Confirmed] 商品识别、推荐算法和知识图谱问答已分别在零售、内容分发和智能客服领域形成较成熟的技术路线。现有研究表明，目标检测模型适合处理商品图片中的多目标识别问题，深度兴趣网络适合建模用户行为序列，图数据库则适合表达商品、类目、品牌和属性之间的结构化关系。

[Inferred] 结合现有 research 结果，本系统并非提出全新算法，而是面向垂直电商业务，将成熟技术进行工程化集成，并在统一业务链路下实现协同应用。

### 1.4 研究内容与论文结构

[Confirmed] 本文主要完成以下工作：

1. 设计 YQSX 智能零食商城的整体系统架构；
2. 设计微服务模块、数据库结构与 RESTful API；
3. 说明 YOLOv11 商品识别模块、DIN 推荐模块和 KBQA 问答模块的实现思路；
4. 结合测试计划整理系统功能测试、接口测试、性能测试与 AI 效果测试方案；
5. 按 GB/T 7714 格式给出参考文献。

[Confirmed] 本章阐述了课题背景、研究意义与论文组织方式，为后续系统设计、系统实现与测试分析奠定基础。

## 第2章 相关技术与系统基础

### 2.1 微服务架构与 Spring Cloud 体系

[Confirmed] 系统采用 Spring Boot、Spring Cloud、Spring Cloud Alibaba、Nacos、Gateway、OpenFeign 与 Sentinel 组成的微服务技术栈。各业务能力按服务边界拆分为商品、订单、用户、支付与 AI 服务，服务之间通过接口交互，不直接跨库访问。

[Confirmed] 这种设计有利于降低耦合、提升部署灵活性，并便于对各服务进行独立扩展与容错治理。

### 2.2 前后端分离与 Web 技术栈

[Confirmed] 前端采用 Next.js 16、React 19、TypeScript、Tailwind CSS、shadcn/ui 与 Zustand 构建页面与交互逻辑。系统页面包括登录、首页、商品列表、商品详情、AI 识别、购物车、订单列表、订单详情与个人中心等。

### 2.3 YOLOv11 图像识别技术

[Confirmed] YOLOv11 用于识别用户上传的零食图片。其任务是从输入图片中检测出商品类别、置信度与边界框信息，再将识别结果映射到商品分类与商品列表，实现拍照购物。

[Confirmed] 在已确认的 research 中，YOLOv11 模块服务以 FastAPI 为接口层，以 PyTorch 为推理框架，识别 19 类零食商品。

### 2.4 DIN 深度兴趣网络推荐技术

[Confirmed] DIN 通过注意力机制建模用户历史行为序列与候选商品之间的相关性，适用于点击率预估与个性化商品推荐。系统将推荐服务拆分为召回与精排两个阶段，以提高在线响应能力。

### 2.5 Neo4j 与 KBQA 技术

[Confirmed] 知识图谱问答模块使用 Neo4j 存储商品实体、类目、品牌、口味与属性之间的关系，并基于实体识别、意图识别和 Cypher 查询生成问答结果。

### 2.6 本章小结

[Confirmed] 本章概述了系统实现所依赖的主要技术，为第三章系统设计提供了技术基础。

## 第3章 系统设计

### 3.1 需求分析

[Confirmed] 系统面向智能零食电商场景，核心需求包括用户注册登录、商品浏览搜索、AI 图像识别、个性化推荐、智能问答、购物车、订单、支付与个人中心管理。

[Confirmed] 系统还需满足微服务隔离、接口安全、幂等控制、限流降级与可观测性要求。

### 3.2 系统总体架构设计

[Confirmed] 系统采用“前端展示层—网关层—业务服务层—AI 服务层—基础设施层”的分层微服务架构。前端通过 Gateway 统一访问后端服务，网关负责鉴权、路由与限流，业务服务负责电商核心流程，AI 服务负责识别、推荐和问答，基础设施层提供注册配置、缓存、数据库与监控能力。

【图片引用占位】图3-1 系统总体架构图（对应 `D:\Programming\YQSX\docs\paper-kit\assets\architecture.png`）

### 3.3 系统模块设计

[Confirmed] 系统模块划分如下：

- 网关服务：统一入口、JWT 鉴权、路由转发、限流熔断；
- 商品服务：商品 CRUD、分类管理、库存管理、识别结果匹配；
- 订单服务：订单创建、状态流转、防重复提交、库存扣减；
- 用户服务：注册登录、JWT 生成与校验、用户信息管理；
- 支付服务：支付单创建、幂等控制、支付状态更新；
- YOLOv11 识别服务：图像识别与类别返回；
- DIN 推荐服务：个性化推荐与缓存管理；
- KBQA 问答服务：知识图谱查询与自然语言应答；
- Web 前端：页面展示与交互逻辑。

【图片引用占位】图3-2 模块关系图（对应 `D:\Programming\YQSX\docs\paper-kit\assets\module-diagram.png`）

### 3.4 数据库设计

[Confirmed] 系统采用微服务数据库隔离原则，不同服务分别使用 `shop_product`、`shop_order`、`shop_user`、`shop_payment` 与 Neo4j。核心业务表包括 `t_category`、`t_product`、`t_user`、`t_user_behavior`、`t_user_address`、`t_order`、`t_order_item` 与 `t_payment`。

[Confirmed] 表设计中统一包含逻辑删除、创建时间与更新时间字段。商品表包含分类、价格、库存、品牌、口味与扩展属性；订单表包含订单号、金额、状态、收货信息与幂等 Token；用户行为表用于支持推荐建模。

【图片引用占位】图3-3 数据库 ER 图（对应 `D:\Programming\YQSX\docs\paper-kit\assets\database-er.png`）

### 3.5 API 接口设计

[Confirmed] 系统遵循 RESTful 风格，统一返回 `code`、`msg`、`data` 三段式响应结构。核心接口包括用户登录注册、商品列表与详情、AI 识别、订单创建与查询、支付创建、推荐获取和知识问答等。

[Confirmed] 对需要鉴权的接口统一通过 `Authorization: Bearer <token>` 传递 JWT。

### 3.6 AI 服务集成设计

[Confirmed] 系统通过 HTTP 调用方式集成 Python AI 服务。商品服务在接收到用户上传图片后，将文件转发至 YOLOv11 识别服务，再依据识别类别查询商品分类和推荐结果；订单服务调用 DIN 服务获取推荐列表；前端可调用 KBQA 服务生成智能导购问答。

### 3.7 本章小结

[Confirmed] 本章从需求、架构、模块、数据库、接口与 AI 集成六个方面完成系统设计，为后续实现提供结构化依据。

## 第4章 系统实现

### 4.1 开发环境与技术实现约束

[Confirmed] 系统后端基于 Java 与 Spring 生态实现，AI 服务基于 Python 实现，前端基于 Next.js 实现。实现过程中遵循服务自治、接口清晰、数据库隔离与安全加固原则。

### 4.2 网关与微服务实现

[Confirmed] 网关服务完成统一入口、JWT 校验、路由转发与限流控制。商品、订单、用户与支付服务分别负责各自业务域，并通过 OpenFeign 或 REST 调用进行协作。

### 4.3 前端页面实现

[Confirmed] 前端实现登录、首页、商品列表、商品详情、AI 识别页、购物车、订单列表、订单详情与个人中心等页面。页面采用组件化方式组织，使用状态管理与 API 封装完成与后端交互。

### 4.4 YOLOv11 商品识别模块实现

[Confirmed] YOLOv11 模块以 FastAPI 暴露 `POST /api/recognize` 接口，接收图片文件，完成格式校验、图像预处理、模型推理、后处理与类别映射，输出识别类别、置信度、边界框与处理时间。

### 4.5 DIN 推荐模块实现

[Confirmed] DIN 推荐模块采用召回+精排架构，先按用户历史行为关联类目和全局热门商品构建候选集，再通过 DIN 模型进行打分排序。缓存命中时返回预计算结果，并通过 JSON+HMAC 方式保障缓存安全。

### 4.6 KBQA 智能问答模块实现

[Confirmed] KBQA 模块通过 Neo4j 存储商品知识图谱，使用词典与规则进行实体识别，基于意图分类选择 Cypher 模板并执行查询，最终生成自然语言答案。
